# 基本语法

## 标题语法

哈希标题语法:

* 用#表示，几个#就是几级标题。
* #与标题文本用空格隔开，确保跨平台兼容性。
* 一般标题前后添加空格行，提高可读性。

## 文本格式

在应用文本格式时，应遵循一些关键原则。
首先，格式应有明确目的，避免过度使用。加粗表示强调关键概念，斜体表示引用或细微强调，划线表示修改。
其次，保持格式一致性。在同一文档内，使用相同的符号来创建相同的格式。例如，如果选用星号作为粗体，请贯穿全文使用它们，而不是与下划线混用。
最后，注意格式符号之间的间距。它们应直接接触格式化文本，避免多余空格。

### 加粗字体

双星号（**）或双下划线（__）。两种方法在功能上相同，但建议使用双星号以实现 Markdown 解析器间的兼容性。

### 斜体

使用单个星号（*）或单个下划线（_）组成。与加粗文字类似，建议使用星号以提升兼容性。

### 斜粗体

当需要同时使用加粗和斜体格式时，可以组合三个星号或下划线。这种格式常用于技术文档中突出特别重要的概念或术语。

### 划线

删除线用于表示已删除或不再有效的内容，使用双波浪号（~~）创建。
该功能在 GitHub Flavored Markdown 中被广泛支持，但可能不包含在标准 Markdown 中。

### 下划线

标准Markdown不直接支持下划线格式，但可以通过HTML标签实现：

<u>This is underlined text</u>

## 段落与换行

首先，避免在段落开头添加空格或制表符进行缩进，因为这可能导致不必要的格式化，尤其是代码块。
第二，用空白行分隔段落。这不仅符合Markdown标准，还提升了纯文本的可读性。即使在Markdown编辑器中，清晰的段落分隔也有助于组织和编辑内容。
对于换行，虽然后置双行距是标准方法，但由于大多数编辑者对空格不可见，换行可能被误删。在需要精确控制的场景中，HTML 标签可能是更可靠的选择。<br>

### 创建段落

单按回车键并不会创建新段落。如果两行文本之间没有空行隔，它们会合并到同一段中。

### 创建换行

有时，段落内需要换行，而不是新段落。Markdown 提供了多种方法，但最可靠的是在按下回车前，在行尾加两个或更多空格。
或者用HTML标签<br>

## 列表语法

### 无序列表

无序列表使用星号（*）、加符号（+）或连字符（-）作为标记。这三种在功能上是相同的，但为了保持一致，最好在整个文档中使用相同的标记。
标记后面必须跟一个空格，然后是列表项内容。这个空间是必须的;没有它，文本将无法被识别为列表项。

### 有序列表

有序列表是通过数字和周期创建的。有趣的是，实际数字并不重要——Markdown 会自动生成正确的序列。这使得编辑过程中插入或删除项目变得更加方便。
虽然数字不必是顺序的，但为了代码可读性，建议使用正确的编号，或者干脆用1来表示所有项目。

### 嵌套列表

列表可以嵌套以创建多层信息结构。嵌套通过缩进实现，通常使用四个空格或单个制表符。嵌套列表可以混合有序和无序类型。

### 向列表中添加其他元素

列表项目可以包含各种类型的内容，包括段落、代码块、引号和其他列表。关键是保持正确的缩进，以确保与列表条目文本位置对齐。

## 块引语法

首先，在引用块前后添加空白行，以区分它们与其他内容，提高可读性。
其次，对于较长的引用，可以考虑添加出处。虽然Markdown没有内置的归属语法，但作者信息可以在引用后附加。
最后，避免过度使用块引号。它们应保留给真实引用内容或特别重要的信息，因为过度使用会削弱其影响力。

### 基本引用

块引号通常使用大于符号>的符号，放置在每行引用文本的开头。这种语法在视觉上模仿了电子邮件中的引用格式。
或者，也可以在段落开头使用一个大于符号，但为清晰和一致，建议在每行前加上。

### 嵌套引用

块引号可以通过添加多个大于符号来嵌套，从而创建不同层次的引号。这在回复或评论场景中尤其有用。

### 引用块中添加其他元素

块引号还可以包含其他Markdown元素，如标题、列表、代码块等，使其高度多样化。

## 链接与图片

### 链接语法

#### 内联链接

内联链接是最常用的类型，语法为 [link text](URL "optional title") 。链接文本会显示给用户，URL是目标地址，悬停时显示可选标题。

#### 参考链接

参考链接将链接定义与使用方式分开，使文档更易于维护，尤其是在同一链接被多次使用时。它们由两部分组成：链接使用和链接定义。
链接定义可以放置在文档中的任意位置，但通常位于段落末尾或文档末尾。

This is a reference link to [Google][google].
You can also write it as [Google][] if the link identifier matches the display text.

[google]: https://www.google.com "Google"

#### 自动链接

对于网址和电子邮件地址，Markdown 提供自动链接功能。只需用<>括号内附上网址或电子邮件地址，即可自动创建链接。

### 图像语法

图像语法与链接语法非常相似，只是在开头加上感叹号。图像语法格式为 ![alt text](image URL "optional title")

存在内联图片与参考图片。

## 代码语法

### 内联代码

内联代码用于将代码嵌入段落中的短代码片段，方法是将代码包裹在回溯引号（'）中。

Use the `console.log()` function to output information to the console.
如果代码本身包含回溯引号，使用多个回溯引号来包裹它：

`` This code contains a ` backtick ``

### 代码块

码块用于显示多行代码。主要有两种方法：缩进代码块和有围栏代码块。

#### 缩进代码块

通过在每行代码开头添加四个空格或一个制表符来创建代码块。

#### 围栏代码块

使用三个回溯跳号（```）或三个波浪号（~~~）来创建一个代码块。这种方法更灵活：

```
function hello() {
    console.log("Hello, World!");
}
```

围栏代码块支持语法高亮，方法是在开头回溯代码后指定语言名称：

~~~javascript
function hello() {
  console.log("Hello, World!");
}
~~~

## 水平线

要创建水平线，请在单独的一行上使用三个或更多星号 (***)、破折号 (---) 或下划线 (___)。
为了兼容性，在水平线前后添加空行。

***

---

___

# 拓展语法

## 表格

要添加表格，请使用三个或更多连字符 (---) 创建每个列的标题，并使用竖线 (|) 分隔每个列。为了兼容性，您还应该在行的两端添加一个竖线。

| Syntax      | Description |
| ----------- | ----------- |
| Header      | Title       |
| Paragraph   | Text        |

可以通过在标题行的连字符左侧、右侧或两侧添加冒号 (:) 将列中的文本左对齐、右对齐或居中对齐。

| Syntax      | Description | Test Text     |
| :---        |    :----:   |          ---: |
| Header      | Title       | Here's this   |
| Paragraph   | Text        | And more      |

## 脚注

脚注允许您添加注释和参考，而不会使文档正文杂乱。

创建脚注时，您添加脚注参考的位置将出现带链接的上标数字。读者可以单击链接跳转到页面底部的脚注内容。

要创建脚注参考，请在方括号内添加插入符号和标识符 ([^1])。标识符可以是数字或单词，但不能包含空格或制表符。标识符仅将脚注参考与脚注本身关联——在输出中，脚注按顺序编号。

使用方括号内的另一个插入符号和数字以及冒号和文本添加脚注 ([^1]: My footnote.)。您不必将脚注放在文档末尾。您可以在列表、块引用和表格等其他元素之外的任何位置放置它们。

**Here's a simple footnote,[^1] and here's a longer one.[^bignote]**

[^1]: This is the first footnote.

[^bignote]: Here's one with multiple paragraphs and code.

    Indent paragraphs to include them in the footnote.

    \`{ my code }\`

    Add as many paragraphs as you like.

## 定义列表

一些 Markdown 处理器允许您创建术语及其相应定义的定义列表。要创建定义列表，请在第一行键入术语。在下一行，键入冒号，后跟空格和定义。

First Term
: This is the definition of the first term.

Second Term
: This is one definition of the second term.
: This is another definition of the second term.

HTML 如下所示

<dl>
  <dt>First Term</dt>
  <dd>This is the definition of the first term.</dd>
  <dt>Second Term</dt>
  <dd>This is one definition of the second term. </dd>
  <dd>This is another definition of the second term.</dd>
</dl>

呈现的输出如下所示

第一个术语

: 这是第一个术语的定义。

第二个术语

: 这是第二个术语的一个定义。 : 这是第二个术语的另一个定义。

## 任务列表

任务列表（也称为检查表和待办事项列表）允许您创建带有复选框的项目列表。
在支持任务列表的 Markdown 应用程序中，复选框将显示在内容旁边。
要创建任务列表，请在任务列表项目前面添加破折号 (-) 和带有空格的方括号 ([ ])。
要选择一个复选框，请在方括号之间添加一个 x ([x])。

* [ ] 一
* [x] 二

- [ ]

+ [ ]

## 表情符号

在大多数情况下，你可以直接从 [Emojipedia](https://emojipedia.org/) 等来源复制一个表情符号，并将其粘贴到文档中。许多 Markdown 应用程序会自动在 Markdown 格式的文本中显示表情符号。从 Markdown 应用程序导出的 HTML 和 PDF 文件应显示表情符号。

一些 Markdown 应用程序允许你通过键入表情符号简码来插入表情符号。这些简码**以冒号开头和结尾**，并包含表情符号的名称。

Gone camping! :tent: Be back soon.

That is so funny! :joy:

## 高亮

这种情况并不常见，但一些 Markdown 处理器允许你高亮文本。结果看起来像这样。要高亮单词，请在单词前后使用两个等号 (==)。

I need to highlight these ==very important words==.

呈现的输出如下所示

我需要高亮这些 ==非常重要的单词== 。

或者，如果你的 Markdown 应用程序支持 HTML，你可以使用 mark HTML 标记。

I need to highlight these <mark>very important words</mark>.

## 下标

这种情况并不常见，但一些 Markdown 处理器允许你使用下标将一个或多个字符置于正常文本行的下方。要创建下标，请在字符前后使用一个波浪号 (~)。

提示
在使用之前，请务必在 Markdown 应用程序中进行测试。一些 Markdown 应用程序在单词前后使用一个波浪号，不是用于下标，而是用于**删除线**。

Docusaurus 中单个波浪号就是删除线。

或者，如果你的 Markdown 应用程序支持 HTML，你可以使用 sub HTML 标记。

H<sub>2</sub>O

## 上标

这并不常见，但一些 Markdown 处理器允许你使用上标将一个或多个字符放置在正常文本行上方。要创建上标，请在字符前后使用一个插入符号 (^)。

或者，如果你的 Markdown 应用程序支持 HTML，你可以使用 sup HTML 标签。

X<sup>2</sup>

## 自动 URL 链接

许多 Markdown 处理器会自动将 URL 转换为链接。这意味着如果你键入 http://www.example.com，你的 Markdown 处理器会自动将其转换为一个链接，即使你没有使用方括号<>。

## 禁用自动 URL 链接

如果你不希望 URL 被自动链接，你可以通过 使用反引号`将 URL 表示为代码 来移除链接。

`http://www.example.com`
